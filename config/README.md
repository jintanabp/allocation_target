# การตั้งค่าและแพ็ก demo

## ไฟล์ `.env`

แนะนำให้วาง **`config/.env`** (คัดลอกจาก **`config/.env.example`** แล้วกรอกค่า) เพื่อรวมใน zip แจกทีมได้ชัดเจน — backend โหลด `config/.env` ก่อน แล้วโหลด `.env` ที่รากโปรเจกต์ถ้ามี (ค่าที่รากจะทับค่าซ้ำ)

## ความปลอดภัย

ไฟล์ `.env` มักมี **client secret** และ ID ภายใน — ถ้า zip แจก ควร **หมุน secret / ตรวจสิทธิ์** หลัง demo และส่งเฉพาะกับผู้ที่จำเป็น
