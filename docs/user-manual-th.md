# คู่มือการใช้งาน Target Allocation

**ระบบกระจายเป้าหมายยอดขาย (หีบ) ให้พนักงานขายรายคน**

เอกสารนี้เขียนสำหรับ Supervisor / Manager ที่ใช้งานแอปบนเบราว์เซอร์  
อ่านตามลำดับแล้วทำตามได้ทีละขั้น — ไม่ต้องอัปโหลดไฟล์ Excel เอง ระบบดึงข้อมูลจากข้อมูลกลางอัตโนมัติ

---

## 1. แอปนี้ทำอะไร

Target Allocation ช่วย **แบ่งจำนวนหีบ (เป้าหมายยอดขาย)** ของแต่ละสินค้า (SKU) ให้พนักงานในทีม โดยอิงจาก:

- **ประวัติการขายย้อนหลัง** — ใครขาย SKU นั้นเยอะ ได้หีบ SKU นั้นเยอะ
- **เป้าเงินรายคน** — มูลค่ารวมต่อคนควรใกล้เป้าที่กำหนด (ยอมคลาดประมาณ ±1,000 บาท/คน)

**ลำดับการใช้งาน (4 ขั้นหลัก):**

1. **ข้อมูลตั้งต้น** — ดูเป้ารวม พนักงาน และ SKU ของงวด
2. **กำหนดเป้าหมาย** *(ข้ามได้)* — ปรับเป้าเงินรายคนถ้าต้องการ
3. **กระจายหีบ** — เลือกวิธีแบ่ง แล้วกดคำนวณ
4. **ส่งผล** — ดาวน์โหลด Excel หรือส่งเข้า Target Sun

---

## 2. ก่อนเริ่มใช้งาน

### สิ่งที่ต้องมี

| รายการ | รายละเอียด |
|--------|------------|
| URL แอป | ได้จากทีม IT / หัวหน้างาน (เปิดบน Chrome หรือ Edge แนะนำ) |
| บัญชี Microsoft | อีเมลองค์กรที่ลงทะเบียนในระบบสิทธิ์แล้ว |
| สิทธิ์ Supervisor | รหัสทีมที่คุณดูแลได้ — แสดงใน dropdown หลังล็อกอิน |
| ข้อมูลเป้างวด | ระบบเป้ากลาง (TGA / Target Sun) ต้องมีเป้าหีบและเป้าเงินของงวดที่เลือก |

### สิ่งที่ไม่ต้องทำ

- ไม่ต้องอัปโหลดไฟล์ Excel เป้าหมาย
- ไม่ต้องติดตั้งโปรแกรมบนเครื่อง — ใช้ผ่านเบราว์เซอร์อย่างเดียว

### อ่านคู่มือก่อนล็อกอิน (แนะนำ)

บนหน้า **เข้าสู่ระบบ** มีปุ่ม **อ่านคู่มือก่อนใช้งาน** (ป้าย **แนะนำ**) — กดได้ก่อนล็อกอิน · สรุป 9 ขั้น ~4 นาที  
หลังเข้า Dashboard ยังมีปุ่ม **คู่มือ** มุมขวาบนตลอดเวลา

---

## 3. เข้าสู่ระบบ

### ทำอย่างไร

1. เปิด URL แอปที่ทีมแจ้ง
2. (แนะนำ) กด **อ่านคู่มือก่อนใช้งาน** บนการ์ดเข้าสู่ระบบ — อ่านได้ก่อนล็อกอิน · หรือกดปุ่ม **คู่มือ** มุมขวาบน
3. กด **ล็อกอินด้วย Microsoft** — ใช้บัญชีองค์กร
4. รอ dropdown **ผู้รับผิดชอบ (Supervisor / Manager)** โหลดเสร็จ
5. เลือกรหัสทีมที่ต้องการกระจายเป้า
6. เลือก **งวดเดือนที่จะกระจายเป้า** (ค่าเริ่มต้นมักเป็นเดือนถัดจากวันนี้)
7. กด **เข้าสู่ระบบ Dashboard**

### ควรเห็นอะไร

- หน้า Dashboard โหลดข้อมูล — อาจใช้เวลาสักครู่
- แถบบนแสดงชื่อแอป และปุ่ม **กลับเลือก Supervisor** / **ออกจากระบบ**

### ถ้าไม่ตรง

| ปัญหา | แนวทาง |
|-------|--------|
| Dropdown Supervisor ว่าง | กด **↻ รีเฟรชรายการ** หรือติดต่อ IT ตรวจสิทธิ์ ACC |
| ล็อกอินไม่ได้ | ตรวจว่าใช้อีเมลองค์กร · ติดต่อ IT เรื่อง Redirect URI |
| ไม่เห็นทีมที่ต้องการ | สิทธิ์อาจจำกัดตาม ACC — ติดต่อหัวหน้าหรือ IT |

---

## 4. ขั้นที่ 1 — ข้อมูลตั้งต้น

### ทำอย่างไร

1. ดู **เป้ารวม** ด้านบน — ยอดเงินรวมของงวดที่เลือก
2. ตรวจตาราง **พนักงาน** — รหัส S/M, ชื่อ, เป้าเริ่มต้น
3. ตรวจตาราง **SKU (เป้าหีบ)** — รหัสสินค้า, แบรนด์, จำนวนหีบเป้ารวมต่อ SKU
4. สลับแท็บ **เทียบเฉลี่ย 3 เดือน** / **เทียบปีที่แล้ว** เพื่อดูบริบทเป้า
5. จัดกลุ่ม SKU เป็น **ราย SKU · แบรนด์ · Section** ตามต้องการตรวจ

### ควรเห็นอะไร

- เป้ารวมเป็นตัวเลขชัดเจน (ไม่ใช่ "—")
- มีรายชื่อพนักงานและ SKU ที่มีเป้าหีบ > 0

### ถ้าไม่ตรง

- **เป้ารวมผิดหรือว่าง** — กดลิงก์ **ติดต่อ IT** ใต้เป้ารวม
- **SKU ไม่ครบ** — ตรวจว่าเลือกงวดถูกต้อง · โหลดหน้าใหม่ (F5)

---

## 5. ขั้นที่ 2 — กำหนดเป้าหมาย (ไม่บังคับ)

ขั้นนี้ **ข้ามได้** — ถ้าใช้เป้า Target Sun ตามที่ระบบดึงมาโดยไม่แก้

### ทำอย่างไร

1. ดูคอลัมน์ **เป้าหมายที่กำหนดเอง** — ค่าเริ่มต้นเท่าเป้า Target Sun
2. คลิกช่องแล้วพิมพ์ปรับเป้าเงินรายคน (ถ้าต้องการ)
3. ระบบคำนวณ **% เติบโต** ให้อัตโนมัติ
4. ตรวจ **ยอดรวมเป้าที่กำหนดเอง** ด้านล่าง — ควรใกล้เป้ารวม

### เกณฑ์ยอดรวม

| ส่วนต่างจากเป้ารวม | ความหมาย |
|---------------------|----------|
| ไม่เกิน ~10 บาท | พร้อมกด **เริ่มคำนวณ** (ขั้น 3) |
| ไม่เกิน ~99 บาท | แจ้งเตือน แต่ยังกดคำนวณได้ |
| มากกว่านั้น | ปุ่ม **เริ่มคำนวณ** ปิด — ต้องปรับให้รวมใกล้เป้ารวม |

### ตัวเลือกเพิ่ม

- **รีเซ็ตเป็น Target Sun** — คืนค่าเป้าที่กำหนดเองทุกคน
- **หักบิวเทรี่ยม** — เปิดช่องหักยอดจากยอดปีที่แล้วก่อนคิด % เติบโต (ใส่เฉพาะคนที่ต้องหัก)

### กรณีเติบโตติดลบ

ถ้าเป้าที่ตั้งทำให้ **% เติบโตติดลบ** (เป้าน้อยกว่ายอดเดือนเดียวกันปีที่แล้ว) ระบบจะขอ **เหตุผล** อย่างน้อย 8 ตัวอักษร ก่อนกดคำนวณ

> ถ้าเป้าเท่า Target Sun แล้วเติบโตติดลบอยู่ดี — ไม่ต้องกรอกเหตุผล

### ถ้าปรับเป้าเงิน

- ถ้าลดเป้าคนหนึ่ง ควร **เพิ่มให้คนอื่น** เพื่อให้ผลรวมยังใกล้เป้ารวม
- เป้าเงินที่ตั้งต่ำกว่าที่ประวัติ×ราคาสะท้อน อาจทำให้หลังคำนวณ **มูลค่ารวมต่อคนยังเกิน/ขาดเป้า** แม้หีบต่อ SKU จะถูกจำกัด ±20% แล้ว — เป็นเรื่องปกติ ควรตรวจว่าเป้าเงินสมเหตุสมผล

---

## 6. ขั้นที่ 3 — กระจายหีบ

### เลือกวิธีกระจาย

เลือกได้ **มากกว่า 1 วิธี** (ติ๊กการ์ดด้านบน):

| วิธี | ความหมาย |
|------|----------|
| **ยอดขายเฉลี่ย 3 เดือนย้อนหลัง (L3M)** | แบ่งตามสัดส่วนยอด 3 เดือน — ค่าเริ่มต้น |
| **ยอดขายเฉลี่ย 6 เดือนย้อนหลัง (L6M)** | แบ่งตามสัดส่วนยอด 6 เดือน |
| **เดือนเดียวกันปีที่แล้ว (LY)** | แบ่งตามยอดเดือนเดียวกันปีก่อน |
| **ผลักดันพนักงาน (PUSH)** | เน้นคนที่ขายน้อย — ได้สัดส่วนมากขึ้น |

ถ้าเลือก **หลายวิธี** — กำหนด **แบรนด์ → วิธี** ใน panel ด้านล่าง

### ตัวเลือกเพิ่ม (checkbox)

- **บังคับให้ทุกคนได้อย่างน้อย 1 หีบ (ต่อ SKU)** — เมื่อเป้าหีบ SKU นั้น ≥ จำนวนพนักงาน
- **SKU ใหม่แบ่งเท่ากัน** — SKU ที่ทั้งทีมไม่มียอด 12 เดือน แบ่งเท่าทุกคน

### หลักการคำนวณ (L3M / L6M / LY)

ระบบทำงานดังนี้:

1. **คำนวณสัดส่วนหีบ** ตามวิธีที่เลือก (เช่น ใครขาย SKU 208405 มาก ได้หีบ SKU นั้นมาก)
2. **จำกัดการปรับ** — หีบต่อ (คน × SKU) ไม่เกิน **±20% จากประวัติเก่า** (สัดส่วนที่ระบบคำนวณตามวิธีที่เลือก)
3. **ปรับเป้าเงินรายคน** — ภายในช่วง ±20% นั้น พยายามให้มูลค่ารวมต่อคนใกล้เป้า (ยอมคลาด ±1,000 บ./คน)

หลังคำนวณ ดูแถบ **📐 หีบ vs ประวัติเก่า (±20%)** เหนือตารางผล — กดกรอง SKU ที่มี ◆ / ⚠ ได้ (รายละเอียดในขั้น 7)

**ตัวอย่าง:** ถ้าสัดส่วนที่คำนวณให้ S003 ได้ 500 หีบของ SKU หนึ่ง — ระบบจะให้ได้ประมาณ **400–600 หีบ** (ไม่ใช่หลักหน่วยถ้าประวัติเป็นหลักร้อย)

### ทำอย่างไร

1. เลือกวิธีกระจาย (และแบรนด์→วิธี ถ้าเลือกหลายวิธี)
2. ติ๊ก checkbox ตามต้องการ
3. ตรวจว่ายอดรวมเป้าเงินขั้น 2 พร้อม (ปุ่ม **เริ่มคำนวณ** เปิด)
4. กด **เริ่มคำนวณ** — รอ progress 4 ขั้น
5. เมื่อเสร็จ ปุ่มเปลี่ยนเป็น **คำนวณใหม่** (ใช้เมื่อเปลี่ยนวิธีหรือแก้เป้า)

## 7. อ่านตารางผลลัพธ์

หลังคำนวณ จะเห็นบล็อก **ผลลัพธ์การกระจายหีบ** พร้อมตัวเลือกเรียง / กรองแบรนด์ และตารางด้านล่าง

### แถบควบคุม (เหนือตาราง)

| ตัวเลือก | ใช้ทำอะไร |
|----------|-----------|
| **เรียงตาม รหัสสินค้า / แบรนด์ / จำนวนหีบ / ราคา** | จัดลำดับคอลัมน์ SKU |
| **ทุกแบรนด์ (ทั้งหมด)** | กรองดูเฉพาะแบรนด์หนึ่ง |
| **บันทึกร่าง · Undo · Excel · ส่ง Target Sun** | เก็บผล / ย้อนแก้ / ส่งออก |

### แถบ 📐 หีบ vs ประวัติเก่า (±20%)

แถบนี้อยู่ **ใต้ปุ่มควบคุม เหนือตาราง** — แสดงทุกครั้งหลังคำนวณ

| ปุ่ม | ความหมาย | การใช้งาน |
|------|----------|-----------|
| **◆ … (ตัวเลข)** | จำนวนช่องที่ **อยู่ในช่วง ±20% ของประวัติเก่า** | **คลิก** เพื่อแสดงเฉพาะคอลัมน์ SKU ที่มี ◆ · คลิกซ้ำหรือ **แสดงทั้งหมด** เพื่อยกเลิกกรอง |
| **⚠ … (ตัวเลข)** | จำนวนช่องที่ **เกินช่วง ±20% ของประวัติเก่า** | **คลิก** เพื่อแสดงเฉพาะ SKU ที่มี ⚠ (มักจากการแก้มือ) |

- ถ้าตัวเลขเป็น **0** ปุ่มจะกดไม่ได้
- ถ้า **ทั้ง ◆ และ ⚠ เป็น 0** จะขึ้นข้อความ ✓ *ไม่พบช่องที่มี ◆ หรือ ⚠ — หีบสอดคล้องประวัติเก่าภายใน ±20%* (ปกติหลังคำนวณ)
- กรองทำงานร่วมกับ **แบรนด์ที่เลือก** ได้ — ถ้าไม่เจอ SKU ในมุมมองนั้น ให้เปลี่ยนแบรนด์หรือกดแสดงทั้งหมด

### โครงสร้างตาราง

| ส่วน | ความหมาย |
|------|----------|
| **แถว** | พนักงาน (S001, S002, …) |
| **คอลัมน์** | SKU แต่ละตัว — หัวคอลัมน์มีรหัส, แบรนด์, ราค/หีบ |
| **ตัวเลขสีน้ำเงิน (ใหญ่)** | จำนวนหีบที่ได้ — **คลิกแก้ได้** |
| **ข้อความสีเทาใต้ตัวเลข** | ประวัติ: เฉลี่ย 3M, เดือนที่แล้ว, เดือนเดียวกันปีก่อน |
| **คอลัมน์ขวาสุด** | รวมหีบ + **มูลค่ารวม** ทุกแบรนด์ |

### มูลค่ารวมต่อคน (คอลัมน์ขวา)

- **✓ ใกล้เป้า** — ห่างจากเป้าเงินไม่เกิน ±1,000 บ.
- **ขาด / เกิน X บาท** — ยังห่างจากเป้าเงินมากกว่า ±1,000 บ.

> บางคนอาจยังเกิน/ขาดเป้าเงินมาก แม้หีบต่อ SKU อยู่ใน ±20% แล้ว — มักหมายความว่า **เป้าเงินที่ตั้ง (ขั้น 2) ไม่สอดคล้องกับประวัติ×ราคา** ไม่ใช่ข้อผิดพลาดของระบบ

### สัญลักษณ์ ◆ และ ⚠

| สัญลักษณ์ | ความหมาย | ควรทำอย่างไร |
|-----------|----------|----------------|
| **◆** (ส้ม) | **อยู่ในช่วง ±20% ของประวัติเก่า** — แสดงในเซลล์และในแถบกรองด้านบน | ปกติเมื่อระบบปรับให้เป้าเงินใกล้ขึ้น — ไม่ต้องแก้ถ้ายอมรับได้ |
| **⚠** (แดง) | **เกินช่วง ±20% ของประวัติเก่า** | มักเกิดจาก **การแก้ตัวเลขเอง** — กดกรอง ⚠ ในแถบด้านบนเพื่อดู SKU ที่เกี่ยวข้อง |

สัญลักษณ์จะอัปเดตอัตโนมัติเมื่อแก้หีบหรือระบบเกลี่ยส่วนต่างหีบ

### แถวล่างตาราง

- **เป้ารวม (หีบ)** — เป้าหีบรวมต่อ SKU จากหัวหน้า
- **รวมพื้นที่จัดสรร** — ผลรวมหีบที่แบ่งให้พนักงาน
- **เครื่องหมาย ✓ สีเขียว** — รวมหีบตรงเป้า SKU นั้น (ควรเป็น ✓ ทุกคอลัมน์ก่อนส่ง Target Sun)

### แก้ไขและ Undo

1. **แก้หีบ** — คลิกตัวเลขสีน้ำเงิน พิมพ์จำนวนใหม่ แล้วคลิกนอกช่อง
2. **↩️ Undo** — ย้อนการแก้ล่าสุด
3. **💾 บันทึกร่าง** — เก็บผลในเบราว์เซอร์ (ไม่แทนการส่ง Target Sun)

---

## 8. ส่งผล — Excel และ Target Sun

### ดาวน์โหลด Excel (สรุปผล Dashboard)

1. กด **↓ ดาวน์โหลด Excel**
2. เลือกแบรนด์ (หรือ ALL)
3. ได้ไฟล์สรุปผลสำหรับเก็บ/ตรวจสอบ

### ส่งเข้า Target Sun

1. กด **📤 ส่งเข้า Target Sun**
2. ใน modal เลือก **ส่งเข้า Target Sun** — ระบบสร้างไฟล์และส่งให้อัตโนมัติ (ไม่ต้องแนบไฟล์)
3. หรือเลือก **ดาวน์โหลด Excel อย่างเดียว** — รูปแบบ TGA สำหรับส่งด้วยตนเอง

### ก่อนส่ง ควรตรวจ

- [ ] คำนวณเสร็จแล้ว (มีตารางผล)
- [ ] แถวล่าง **รวมพื้นที่จัดสรร** ตรง **เป้ารวม (หีบ)** ทุก SKU (✓)
- [ ] แก้มือแล้วกดกรอง **⚠** ในแถบด้านบน ตรวจ SKU ที่เกินช่วง ±20%

### ใครกดส่งได้

## 9. คำถามที่พบบ่อย (FAQ)

**ถาม: ไม่เห็นแถบ ◆ / ⚠ หรือตัวเลขเป็น 0 ทั้งคู่?**  
ตอบ: ปกติหลังคำนวณ — หีบอยู่ใน ±20% ของประวัติเก่า · แถบยังแสดงอยู่เหนือตารางพร้อมข้อความ ✓ · ถ้าแก้หีบมือแล้วเกิน ±20% ตัวเลข ⚠ จะขึ้นและกดกรองได้

**ถาม: กดกรอง ◆ / ⚠ แล้วตารางว่าง?**  
ตอบ: ลองเปลี่ยนจากแบรนด์เดียวเป็น **ทุกแบรนด์** หรือกด **แสดงทั้งหมด** ในแถบกรอง

**ถาม: ต้องทำขั้น 2 ทุกครั้งไหม?**  
ตอบ: ไม่ — ข้ามได้ ใช้เป้า Target Sun ตามที่ระบบดึงมา

**ถาม: ทำไมหีบไม่ตรงกับเฉลี่ย 3M ในเซลล์พอดี?**  
ตอบ: ระบบแบ่งตามสัดส่วน 3M เป็นหลัก แต่อาจปรับภายใน ±20% เพื่อให้เป้าเงินรายคนใกล้ขึ้น — ดูสัญลักษณ์ ◆

**ถาม: ◆ เยอะ ผิดปกติไหม?**  
ตอบ: ไม่จำเป็นผิด — แปลว่าหีบยังอยู่ในช่วง ±20% ของประวัติเก่า แต่ระบบปรับเพื่อเป้าเงิน ถ้ายอมรับได้ไม่ต้องแก้

**ถาม: เห็น ⚠ หลังแก้มือ ทำอย่างไร?**  
ตอบ: ถ้าแก้ตั้งใจ — ไม่เป็นไร แต่ควรตรวจว่าผลรวมหีบต่อ SKU ยังตรงเป้า (แถวล่าง ✓)

**ถาม: เป้าเงินบางคนยังเกิน/ขาดมาก ทั้งที่ ◆ แล้ว?**  
ตอบ: เป้าเงินขั้น 2 อาจไม่สอดคล้องกับประวัติ×ราคา — ตรวจ/ปรับเป้าเงินรายคน หรือยอมรับว่าทั้งสองอย่างพร้อมกันไม่ได้

**ถาม: กดเริ่มคำนวณไม่ได้**  
ตอบ: ตรวจยอดรวมเป้าเงินขั้น 2 ใกล้เป้ารวมหรือไม่ · กรอกเหตุผลเติบโตติดลบ (ถ้าระบบขอ)

**ถาม: เป้ารวมขั้น 1 ผิด**  
ตอบ: กด **ติดต่อ IT** — อย่าบังคับคำนวณด้วยเป้าที่ผิด

**ถาม: ปุ่มส่ง Target Sun สีเทา**  
ตอบ: ยังไม่คำนวณ · หรือไม่มีสิทธิส่ง — ติดต่อ IT

**ถาม: ส่งแล้ว error / timeout**  
ตอบ: ลองใหม่ · ติดต่อ IT ถ้า UAT/เครือข่ายมีปัญหา

**ถาม: บันทึกร่าง vs ส่ง Target Sun ต่างกันอย่างไร?**  
ตอบ: บันทึกร่างเก็บในเบราว์เซอร์เครื่องนี้เท่านั้น — ส่ง Target Sun คือส่งเข้าระบบเป้าจริง

**ถาม: เปลี่ยน Supervisor / งวด ทำอย่างไร?**  
ตอบ: กด **กลับเลือก Supervisor** ที่แถบบน แล้วเลือกใหม่

**ถาม: คู่มือในแอปอยู่ที่ไหน?**  
ตอบ: หน้า Login — ปุ่ม **อ่านคู่มือก่อนใช้งาน** บนการ์ดเข้าสู่ระบบ (อ่านได้ก่อนล็อกอิน) · มุมขวาบนมีปุ่ม **คู่มือ** ตลอดเวลา — สรุปสั้น 9 ขั้น

## 10. คำศัพท์สั้น ๆ

| คำ | ความหมาย |
|----|----------|
| **Supervisor / Manager** | รหัสผู้รับผิดชอบทีมขาย — เลือกตอนล็อกอิน |
| **SKU** | รหัสสินค้า — แต่ละคอลัมน์ในตารางผล |
| **หีบ** | หน่วยเป้าหมายยอดขายที่แบ่งให้พนักงาน |
| **เป้ารวม** | ยอดเงินเป้ารวมของทีมในงวด |
| **เป้า Target Sun / เป้าที่กำหนดเอง** | เป้าเงินรายคนจากระบบ หรือที่ปรับในขั้น 2 |
| **L3M / L6M / LY** | วิธีแบ่งตามประวัติ 3 เดือน / 6 เดือน / เดือนเดียวกันปีก่อน |
| **ช่วง ±20% / ประวัติเก่า** | หีบต่อ (คน×SKU) เปรียบเทียบกับสัดส่วนที่ระบบคำนวณจากประวัติ — ไม่เกิน ±20% (ยกเว้นแก้มือจนเกิน → ⚠) |
| **◆** | อยู่ในช่วง ±20% ของประวัติเก่า |
| **⚠** | เกินช่วง ±20% ของประวัติเก่า — มักจากการแก้มือ |
| **Target Sun** | ระบบเป้ายอดขายกลาง — ปลายทางหลังกระจายหีบ |
